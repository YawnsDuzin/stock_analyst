## 7.1 개발 전략: 바이브 코딩이란?

바이브 코딩(Vibe Coding)은 AI와 협업하며 빠르게 프로토타입을 만들어가는 개발 방식입니다.

### 핵심 원칙

1. **작은 단위로 빠르게**: 한 번에 하나의 기능만 구현
2. **즉시 테스트**: 구현 즉시 동작 확인
3. **AI 활용 극대화**: Claude에게 코드 생성, 리뷰, 디버깅 요청
4. **점진적 개선**: 처음부터 완벽을 추구하지 않음

### 개발 순서

```
PRD 작성 (Claude에게 요청)
  → 기본 구조 생성 (Claude 코드 생성)
  → 핵심 기능 구현 (반복적 대화)
  → 테스트 및 디버깅 (Claude 지원)
  → 배포 및 모니터링
  → 피드백 반영 → 다음 Phase
```

## 7.2 Phase 1 - MVP 구축

### 목표

DART + Claude로 기본 분석 리포트를 자동 생성하고, 텔레그램으로 알림을 받는 최소 기능 시스템

### 구현 항목

```
1. FastAPI 기본 구조 생성
   ├── 프로젝트 스캐폴딩
   ├── 설정 관리 (pydantic-settings)
   └── 기본 API 엔드포인트

2. DART API 연동
   ├── DART API 키 발급 및 설정
   ├── 사업보고서 조회 기능
   ├── 공시 목록 조회 기능
   └── 대량보유/임원매매 보고 조회

3. Claude Tool Use 기본 연동
   ├── Anthropic API 설정
   ├── 기본 분석 프롬프트
   ├── Tool 3개 정의 (DART 조회, 감성 분석, 리포트 생성)
   └── Tool Use 실행 루프

4. 텔레그램 봇 알림
   ├── 봇 생성 및 설정
   ├── 기본 명령어 (/analyze, /status)
   └── 신호 알림 전송

5. SQLite 기본 데이터 저장
   ├── 기업 정보 테이블
   ├── 분석 결과 테이블
   └── 신호 이력 테이블
```

### 산출물

- 특정 종목 입력 → 기본 분석 리포트 생성
- 텔레그램으로 분석 결과 수신
- `/analyze 005930` 명령으로 삼성전자 분석 실행

### Phase 1 디렉토리 구조

```
src/stock_analyst/
├── __init__.py
├── main.py               # FastAPI 앱
├── config.py             # 설정
├── data_sources/
│   ├── __init__.py
│   └── dart_collector.py # DART API
├── agents/
│   ├── __init__.py
│   └── analyst_agent.py  # Claude 연동
├── notifications/
│   ├── __init__.py
│   └── telegram_bot.py   # 텔레그램
└── models/
    ├── __init__.py
    └── database.py       # SQLite
```

## 7.3 Phase 2 - 대안 데이터 파이프라인

### 목표

핵심 대안 데이터 소스 3~4개를 연결하여 자동 신호 스캐너 구현

### 구현 항목

```
1. 나라장터 API 연동
   ├── API 키 발급 (공공데이터포털)
   ├── 낙찰 결과 수집기
   ├── 계약금액/시가총액 비율 계산
   └── 임계값 초과 시 알림 로직

2. 채용공고 크롤러
   ├── 사람인 크롤러
   ├── 잡코리아 크롤러
   ├── 원티드 크롤러
   ├── 주간 스냅샷 저장
   └── 변화량 분석 로직

3. 뉴스 크롤러 + Claude 감성 분석
   ├── 네이버 뉴스 크롤러
   ├── 한경/매경 크롤러
   ├── Claude 감성 분석 프롬프트
   └── 키워드 빈도 추적

4. HIRA 의약품 데이터 (제약 섹터)
   ├── 데이터 다운로드 자동화
   ├── 처방 트렌드 분석
   └── 섹터별 처방 점유율 추적

5. ChromaDB RAG 구성
   ├── 문서 임베딩 파이프라인
   ├── 한국어 임베딩 모델 설정
   └── RAG 검색 기능
```

### 산출물

- 매일 자동으로 유의미한 신호가 감지된 종목 리스트 생성
- 나라장터 대형 계약 발견 시 즉시 알림
- 채용 패턴 변화 주간 리포트

## 7.4 Phase 3 - 자동화 및 대시보드

### 목표

완전 자동화된 데이터 수집/분석 + 실시간 시각화 대시보드

### 구현 항목

```
1. Celery 스케줄러 구성
   ├── Redis 브로커 설정
   ├── 매일 새벽 5시 자동 수집 태스크
   ├── 실시간 공시 감시 태스크
   └── 주간 종합 리포트 태스크

2. AlphaScorer 통합 스코어 계산
   ├── 6개 신호 통합 모듈
   ├── 가중 평균 계산
   ├── 섹터별 가중치 조정
   └── 점수 이력 저장

3. Next.js 대시보드 구현
   ├── 메인 대시보드 (종목 요약)
   ├── 종목 상세 페이지
   ├── 신호 타임라인
   └── 차트 (Recharts / TradingView LWC)

4. Supabase 실시간 연동
   ├── Supabase 프로젝트 설정
   ├── 실시간 구독 (Realtime)
   ├── 사용자 인증
   └── Row Level Security

5. 임계값 초과 종목 자동 알림
   ├── 점수 80+ 즉시 알림
   ├── 급격한 점수 변화 알림
   └── 일간 요약 알림
```

### 산출물

- 실시간 모니터링 대시보드 (웹)
- 완전 자동화된 일간 분석 파이프라인
- 임계값 기반 자동 알림 시스템

## 7.5 Phase 4 - 백테스팅 및 고도화

### 목표

신호의 실제 유효성을 검증하고, 전략을 최적화하는 고도화 단계

### 구현 항목

```
1. 과거 신호 vs 실제 주가 성과 검증
   ├── 과거 데이터 수집 (3년치)
   ├── 신호 시점별 후속 수익률 계산
   ├── Information Coefficient (IC) 측정
   └── 적중률/샤프비율 계산

2. 가중치 최적화 (Optuna 활용)
   ├── 목적 함수 설계 (IC 최대화)
   ├── 500회 이상 시행
   ├── 과적합 방지 (Walk-Forward)
   └── 최적 가중치 적용

3. 멀티 에이전트 구조 전환
   ├── DataCollectorAgent
   ├── AnalystAgent (Claude)
   ├── RiskManagerAgent (Claude)
   └── 에이전트 간 통신 프로토콜

4. 포트폴리오 시뮬레이터 구현
   ├── 가상 포트폴리오 관리
   ├── 리밸런싱 로직
   ├── 성과 측정 (수익률, 샤프, MDD)
   └── 벤치마크 대비 성과 비교
```

### 산출물

- 검증된 알파 전략 (IC, 적중률, 샤프비율 리포트)
- 최적화된 가중치 세트
- 자동 포트폴리오 제안 시스템

## 7.6 PRD 템플릿

바이브 코딩 시작 시 Claude에게 제공할 PRD(Product Requirements Document) 템플릿입니다.

```markdown
# [프로젝트명] PRD

## 1. 목표
- **무엇을 만드는가**: [한 문장으로 설명]
- **왜 만드는가**: [해결하려는 문제]
- **성공 지표**: [측정 가능한 KPI 2~3개]

## 2. 기능 요구사항
### 핵심 기능 (Must-have)
1. [기능 1]: 입력 → 출력 정의
2. [기능 2]: 입력 → 출력 정의
3. [기능 3]: 입력 → 출력 정의

### 보조 기능 (Nice-to-have)
1. [기능 A]
2. [기능 B]

## 3. 기술 스택
- **언어**: Python 3.12+
- **프레임워크**: FastAPI
- **DB**: PostgreSQL / SQLite
- **AI**: Claude API (anthropic)
- **기타**: [추가 라이브러리]

## 4. 데이터 흐름
```
수집 → 처리 → 저장 → 분석 → 출력
```

## 5. API 설계
### POST /api/analyze
- Request: { "ticker": "005930" }
- Response: { "score": 85, "signal": "STRONG_BUY", ... }

## 6. 제약사항
- 예산: API 비용 월 $XX 이내
- 기간: Phase 1 완료 기준
- 기술: Python 생태계 내

## 7. 다음 단계
- Phase 1 완료 기준: [명확한 정의]
- Phase 2 시작 조건: [의존성]
```

### 추천 시작 조합

```
Week 1-2: 나라장터 API + DART 기본 + Claude Tool Use + 텔레그램 알림 (MVP)
Week 3-4: 채용공고 크롤러 + NLP 감성 분석 추가
Week 5-6: AlphaScorer 통합 + Next.js 대시보드
Week 7-8: 백테스팅 + 가중치 최적화
```

## 핵심 요약

```
Phase 1 (MVP)      = DART + Claude + 텔레그램 → 기본 분석 리포트
Phase 2 (파이프라인) = 나라장터 + 채용 + 뉴스 + HIRA → 자동 신호 스캔
Phase 3 (자동화)    = Celery + AlphaScorer + Next.js → 실시간 대시보드
Phase 4 (검증)      = 백테스팅 + Optuna + 시뮬레이터 → 검증된 전략
```
