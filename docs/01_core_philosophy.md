## 1.1 왜 틈새인가?

### 대형 증권사 애널리스트의 한계

대형 증권사 애널리스트는 다음과 같은 구조적 제약을 가지고 있습니다:

1. **커버리지 제한**: 한 명의 애널리스트가 담당하는 기업은 보통 10~20개
2. **컨센서스 중심**: 분기 실적과 EPS 추정에 집중, 이미 시장에 반영된 정보
3. **이해 충돌**: IB(투자은행) 부문과의 이해관계로 솔직한 의견 제시가 어려움
4. **대형주 편중**: 시가총액 상위 종목 중심, 중소형주는 커버리지 사각지대

### 틈새 전략의 본질

```
일반 애널리스트의 세계:
  재무제표 → EPS 추정 → PER 비교 → 목표주가
  (= 모두가 같은 데이터를 보고, 같은 결론에 도달)

이 시스템의 세계:
  숨겨진 공개 데이터 → 선행 신호 발굴 → 시장보다 빠른 판단
  (= 아무도 보지 않는 공개 데이터에서 알파를 추출)
```

**핵심 통찰**: 정보의 가치는 "비밀"에 있는 것이 아니라, "아무도 보지 않는 공개 정보"에 있습니다. 나라장터 낙찰 데이터, 채용공고 변화, 특허 출원 트렌드는 모두 공개 정보이지만, 이를 체계적으로 수집하고 분석하는 사람은 극소수입니다.

## 1.2 3가지 알파 원천

### 알파(Alpha)란?

알파는 시장 평균 수익률을 초과하는 초과 수익을 의미합니다. 이 시스템은 3가지 유형의 비대칭성에서 알파를 발굴합니다.

### 정보 비대칭 (Information Asymmetry)

| 구분 | 설명 | 선행성 | 예시 |
|------|------|--------|------|
| **정보 비대칭** | 공개됐지만 아무도 안 보는 데이터 | 1~4주 | 나라장터 대형 낙찰, 해외 임상 데이터 |

- **메커니즘**: 데이터는 공개되어 있지만, 체계적으로 모니터링하는 참여자가 거의 없음
- **활용법**: 자동화된 데이터 파이프라인으로 24/7 모니터링
- **수익 기회**: IR 공시 전 나라장터에서 대형 계약을 발견하면 2~4주 선행 투자 가능

### 해석 비대칭 (Interpretation Asymmetry)

| 구분 | 설명 | 선행성 | 예시 |
|------|------|--------|------|
| **해석 비대칭** | 데이터는 있지만 잘못 해석되는 경우 | 1~2분기 | 채용공고에서 사업 방향 전환 감지 |

- **메커니즘**: 시장 참여자 대부분이 재무 데이터에만 집중하여, 비재무 신호를 놓침
- **활용법**: NLP 기반 언어 변화 감지, 채용 패턴 분석으로 사업 방향 예측
- **수익 기회**: CEO의 컨퍼런스콜 톤 변화를 감지하여 실적 서프라이즈 사전 예측

### 속도 비대칭 (Speed Asymmetry)

| 구분 | 설명 | 선행성 | 예시 |
|------|------|--------|------|
| **속도 비대칭** | 시장이 반응하기 전 먼저 포지션 | 수일~수주 | 대량보유 보고 변동 조기 감지 |

- **메커니즘**: 정보가 시장에 반영되는 데 시차가 존재
- **활용법**: 실시간 데이터 파이프라인 + 자동 알림으로 반응 속도 극대화
- **수익 기회**: 규제 변화, 정책 발표 시 수혜/피해 섹터를 자동 매핑하여 즉시 대응

## 1.3 시스템의 핵심 원칙

### 원칙 1: 공개 데이터만 활용한다

```
✅ 허용: 나라장터, DART, 공공API, 해외 공시, 뉴스, 채용공고
❌ 금지: 미공개 중요 정보(MNPI), 해킹, 내부자 정보
```

모든 데이터 소스는 합법적으로 접근 가능한 공개 정보여야 합니다. 이는 법적 리스크를 제거하고, 시스템의 지속 가능성을 보장합니다.

### 원칙 2: 반론을 항상 검토한다

모든 투자 가설에 대해 반드시 "이 가설이 틀릴 수 있는 이유"를 함께 검토합니다. Claude에게 Devil's Advocate 역할을 부여하여 편향을 줄입니다.

### 원칙 3: 정량적 근거를 동반한다

"좋아 보인다"가 아니라 "나라장터 계약금액이 시가총액의 15%에 해당하며, 과거 유사 사례에서 평균 12% 주가 상승이 있었다"와 같이 구체적 수치로 판단합니다.

### 원칙 4: 리스크를 먼저 정의한다

포지션 진입 전 반드시 손절 조건과 최대 손실 규모를 사전에 정의합니다.

### 원칙 5: 점진적 확대를 추구한다

신호의 유효성이 백테스팅과 실전 소액 테스트로 검증된 이후에만 포지션 규모를 확대합니다.

## 1.4 일반 애널리스트 vs 이 시스템

| 비교 항목 | 일반 애널리스트 | 이 시스템 |
|-----------|----------------|-----------|
| **데이터 소스** | 재무제표, IR 자료 | 공공 데이터, 대안 데이터, NLP |
| **분석 주기** | 분기별 | 실시간/일간 |
| **커버리지** | 대형주 10~20개 | 전 종목 자동 스캐닝 |
| **반론 검토** | 주관적 | AI 기반 체계적 검증 |
| **선행성** | 공시 후 반응 | 공시 전 신호 감지 |
| **감정 개입** | 있음 | 최소화 (정량적 판단) |
| **확장성** | 인력 한계 | 자동화로 무한 확장 |

## 1.5 왜 AI/Claude가 필요한가

### 데이터 처리 규모의 문제

매일 수천 건의 공시, 수백 건의 채용공고, 수십 개의 해외 데이터가 쏟아집니다. 인간이 이를 모두 읽고 분석하는 것은 불가능합니다. Claude는:

1. **대량 텍스트 처리**: 사업보고서, 컨퍼런스콜 전문을 읽고 핵심 변화를 추출
2. **교차 분석**: 서로 다른 데이터 소스 간의 상관관계를 발견
3. **언어 이해**: 한국어/영어 혼합 데이터를 자연스럽게 처리
4. **반론 생성**: 투자 가설에 대한 체계적인 반론 제시

### Tool Use를 통한 자동화

Claude의 Tool Use 기능을 활용하면, AI가 직접 데이터베이스를 조회하고, API를 호출하며, 분석 결과를 산출할 수 있습니다. 이는 단순한 챗봇이 아닌, 능동적으로 정보를 수집하고 분석하는 에이전트 시스템을 가능하게 합니다.

### RAG를 통한 맥락 강화

ChromaDB 벡터 데이터베이스에 과거 분석 리포트, 공시 내용, 뉴스 기사를 저장하여, Claude가 분석 시 관련 맥락을 자동으로 참조할 수 있습니다.

## 핵심 요약

```
1. 틈새 = 아무도 보지 않는 공개 데이터에서 알파 발굴
2. 3가지 비대칭성 = 정보, 해석, 속도
3. 5가지 원칙 = 공개 데이터, 반론 검토, 정량적 근거, 리스크 선정의, 점진적 확대
4. AI/Claude = 대규모 데이터 처리 + 교차 분석 + 반론 생성의 핵심 엔진
```
